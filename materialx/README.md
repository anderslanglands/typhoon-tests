- `base.usda` contains `/World/Sphere`, a converted `UsdGeomMesh` version of `resources/Geometry/sphere.obj`, including face-varying `primvars:st`; it does not reference the OBJ at render time.
- `base.usda` owns `/World/Camera`, `/World/DomeLight`, `/Render/Settings`, `/Render/Settings/Product`, and `/Render/Settings/Product/Var`.
- `/World/Camera` is authored for a 45 degree square FOV with `focalLength = 50`, `horizontalAperture = 41.421356`, and `verticalAperture = 41.421356`.
- Per-material USDAs sublayer `base.usda`, bind `/World/Sphere`, define materials under `/Looks`, and over `/Render/Settings/Product` to set `productName`.
- `/Render/Settings` authors `int2 resolution = (512, 512)` from `_options.mtlx`.
- Explicit MaterialX `colorspace` attributes are preserved as USD `colorSpace` metadata on generated shader input properties.
- USD `DomeLight` has the radiance HDR applied as `inputs:texture:file`; the MaterialX irradiance HDR path is preserved as `materialx:irradianceIBL:file` provenance metadata.

## Local Asset Copies

Referenced images and HDR maps are copied under `assets/`, and USDA asset paths are rewritten to use those local copies.

## Rendering

Use the root Pixi tasks to render this suite:

```bash
pixi run render-materialx-all
pixi run render-materialx-one materialx/open_pbr_carpaint_Car_Paint.usda
```

The tasks reuse `/home/anders/code/openusd-omniverse/pixi.toml` with `pixi run --manifest-path`, so they run the OpenUSD `usdrender` task directly. The OpenUSD task supplies `--renderer Embree --complexity high`; the local wrapper supplies `--disableCameraLight`, the USDA path, and `--outputRoot`.

Useful environment overrides:

```bash
MATERIALX_RENDER_DRY_RUN=1 pixi run render-materialx-all
MATERIALX_RENDER_FAIL_FAST=1 pixi run render-materialx-all
MATERIALX_RENDER_OUTPUT_ROOT=/tmp/materialx-renders pixi run render-materialx-all
OPENUSD_PIXI=/path/to/openusd/pixi.toml pixi run render-materialx-all
```
